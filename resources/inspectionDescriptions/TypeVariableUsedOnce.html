Reports type variables declared in a type specification head that are referenced only once.

A type variable that appears only in the head of a @type, @typep, @opaque, @spec, @callback, or @macrocallback and is never used in the body constrains nothing. Such a parameter is almost always a mistake - either the body should reference it, or the parameter is unnecessary.

This inspection flags the declaring parameter (for example, a in @type single_type(a) :: integer()) when its name occurs fewer than twice across the whole specification.

The following are not flagged:

  • Type variables referenced at least twice, for example a and b in @type pair(a, b) :: {a, b} or the repeated a in @type twins(a) :: {a, a}.
  • Parameters whose name begins with an underscore (for example _a), which conventionally marks an intentionally unused variable.

To fix a reported type variable, reference it in the body at least once more, or replace it with a concrete type such as term().
