Reports module qualifiers in qualified calls that cannot be resolved to any module definition in scope.

In Elixir, a qualified call like Module.function(args) requires that Module resolves to a module that is defined, aliased, or otherwise in scope. When the qualifier module cannot be resolved, the call will fail at runtime with UndefinedFunctionError.

This inspection flags the qualifier (for example, Unresolved in Unresolved.changeset()) when it does not resolve through any alias directive, direct module definition, or import in the current scope.

Dynamic qualifiers (variables, function call results, module attributes, map/struct field access) are excluded because they cannot be resolved statically.

Fully-qualified Erlang-style module names with the Elixir. prefix (e.g. Elixir.Enum) are recognised: the prefix is stripped and the underlying module is checked for existence.

This inspection only runs on files under Source or Test Source roots (e.g. lib/, test/). Files in unmarked directories like config/ are not inspected.

Files containing use calls whose macro expansion cannot be traced statically (e.g. ExUnit.CaseTemplate) are also excluded, since the use may inject aliases that are invisible to static analysis.
