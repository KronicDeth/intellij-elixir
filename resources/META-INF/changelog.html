v23.1.0

  • Enhancements

    • Debugger now uses structured coroutines with proper lifecycle management, improving stability during session start, stop, and teardown.
    • Debugger operations serialised to prevent concurrent BEAM network calls, reducing timeout errors and improving reliability under load.
    • WSL debugger support: source navigation now resolves Linux-style paths reported by the BEAM to their Windows equivalents, fixing breakpoint hits, stack frame navigation, and failed-breakpoint messages in WSL projects.
    • mix format integration rewritten using modern async formatting APIs. Improved error reporting with clickable error locations and better cancellation handling.
    • ANSI escape codes disabled in formatter subprocess output for reliable error parsing.
    • Breakpoint availability checks optimised to avoid blocking disk reads, reducing UI freezes when setting breakpoints in large projects.
    • SDK setup flows modernised -- progress dialogs now display correctly and no longer block the UI thread.

  • Bug Fixes

    • Fixed EDT blocking during debug sessions caused by network calls running on the UI thread.
    • Fixed UI freezes when setting breakpoints in files with deep module nesting -- directory traversal no longer triggers blocking filesystem reads.
    • Fixed debugger source navigation failing in WSL projects when the BEAM reports Linux-style paths.
    • Fixed Erlang SDK version detection blocking the UI thread without a progress indicator.

v23.0.7

  • Enhancements

    • Unicode identifier and atom support -- syntax highlighting and parsing now works for non-ASCII identifiers and atoms (e.g. Unicode function names, bare Unicode atoms).
    • Decompiled Erlang private functions now shown as defp instead of def, giving an accurate picture of each module's public API.
    • Erlang type signatures (-type/-opaque) decompiled as @type/@opaque with proper rendering.
    • Module resolution now treats Elixir.Enum and Enum identically for completion, hover docs, and Go-to-Declaration.
    • Bulk decompilation no longer blocks the UI and produces cleaner log output with deduplicated warnings.
    • Improved diagnostics when BEAM decompilation encounters parse errors, with summarised error patterns in the run log.

  • Bug Fixes

    • IDE freeze during bulk decompilation on WSL: file handles were leaked, eventually exhausting the connection to the WSL filesystem and freezing the IDE.
    • Indexing deadlock when opening WSL projects -- BEAM file reads no longer block the write-action queue.
    • Numerous decompiler output corrections that previously produced unparseable Elixir, breaking Go-to-Declaration, hover docs, and completion for BEAM dependencies:
      • Block expressions (case/if/try) and nested <<>> literals inside binary specs now parenthesised correctly.
      • Word-based unary operators (not, etc.) now followed by a space.
      • Erlang compiler-generated variable names containing @ sanitised to valid Elixir identifiers.
      • Interpolation markers, backslashes, and Unicode escape sequences properly escaped in decompiled strings and atoms.
      • Carriage returns stripped from BEAM doc strings.
      • Modules compiled without debug info no longer cause SEVERE log floods.
      • Struct function resolution (__struct__/1) now works correctly.
      • Erlang record wildcards handled without log errors.
      • Zero-argument anonymous functions, empty map updates, bitstring generators, and non-integer type specifiers all render correctly.
      • Comprehension generators with do-end blocks decompile correctly.
      • Atoms with invalid alias shapes fall back to quoted form instead of crashing the decompiler.
