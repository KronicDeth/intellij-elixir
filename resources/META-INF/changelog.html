v23.0.5

  • Bug Fixes

    • Fixed Umbrella project import crash when root folder and child app share the same name (e.g., emqx/ root with apps/emqx/ child). The quick import path (File > Open on mix.exs) bypassed the wizard's duplicate detection, causing ModuleWithNameAlreadyExists. Module names are now disambiguated using the relative path (e.g., emqx for the root, emqx-apps-emqx for the child). - @joshuataylor, with thanks to @JiaRG for the thorough and reproducible bug report and excellent example umbrella project!
